## G. Calendar history behavior

pushState: month prev/next, day click, country, city, type, series, status,
reset. replaceState: search typing, initial normalization, popstate restore.
Back/Forward restore exact month + filters + selected day + URL; no per-keystroke
entries; equivalent states de-duplicated.

## H. Month-scoped loading

Server: `loadEventsInRange(from,to)` with overlap predicate
`startDateTime <= to AND (endDateTime >= from OR (no endDateTime AND startDateTime >= from))`,
lightweight `select`, tagged `unstable_cache` (`revalidate: 300`).
Public endpoint `/api/public/events` validates dates, ordering, ≤62-day span,
and locale; returns a typed summary DTO; `Cache-Control: public, s-maxage=300,
SWR`. Client fetches per visible grid range with abort + loading + retryable
error; events page seeds only ~now±window + facets (no 200-event archive shipped).

## I. Detail-route performance before/after

- Before: person detail issued `getEventBySlug`-style person load **plus a full
  `getEvents(locale)` archive load** (up to 200 events) filtered in the render.
- After: person load + one `getEventsBySlugs(person.eventSlugs)` targeted query
  (≤ number of appearances), lightweight projections. Warm requests served via
  ISR/data cache (`revalidate: 300`).
